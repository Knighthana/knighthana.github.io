探寻

起初，我认为这是某种正则表达式，于是以regular expression为关键词进行了大量的查找，然而大多数答案只是说%%在正则表达式中表示一个字面意义的%，并没有找到满意的答案

最后还是得靠manpage：man bash，然后用/搜索%，我很快找到了答案，就在“Parameter Expansion”一节中：

也就是说，这里的%与#是bash解析脚本时使用的，它们作为bash的参数扩展进行使用；

其中，##用于清理前缀，%%用于清理后缀，而且允许搭配*来选中任意内容；

顺便，单井号#和单百分号%也是可以的，但是为了代码的辨识度，还是用双井号与双百分号吧；

关于read的-r选项的解释

-r

Backslash does not act as an escape character. The backslash is considered to be part of the line. In particular, a backslash-newline pair may not be used as a line continuation.

也就是说，用-r时，read会忽略反斜杠的escape含义也就是转义含义，会把反斜杠作为句子的一部分处理；

特别地，即便使用了-r选项，每行输入仍然被视为独立的行，而不能使用反斜杠和换行符来试图将它们组合成单个逻辑行。
